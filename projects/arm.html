Creating Session & Password Generation

Users select a keyword and number, which are then combined with the authenticated user’s internal UID. These values are passed through a one-way hashing mechanism to produce a secure, repeatable password.

Because the system does not store actual passwords, users regenerate them at any time using the same inputs. This approach eliminates the risk associated with database breaches, as only the keyword and number—never the final password—are stored. All user metadata is protected through encrypted Laravel sessions and strict database constraints.

Password Saving & Database Management

When users save an entry, CyberSuite stores the structured components required to later regenerate the password:

  • entry_name – A human-readable label for the account
  • keyword – User-chosen word, stored securely
  • number – User-chosen numerical input
  • user_id – Foreign-key reference to the authenticated user

All entries can be viewed, edited, deleted, or re-generated. The model relationships enforce referential integrity, and the controller handles validation, encryption, and safe database writes.

Security Web Scanner

The CyberSuite Web Scanner performs a multi-layer security assessment that combines static inspection, active probing, cryptographic analysis, and external threat intelligence. When a URL is submitted, the Node.js backend aggregates results from multiple scanners and returns a unified, human-readable security posture report.

Scanner Capabilities

  • Phishing & Malware Detection – Integrates the Google Safe Browsing API to identify malicious, deceptive, or compromised URLs.
  • Domain Intelligence – Collects registrar data, domain age, creation date, and expiration to evaluate legitimacy and detect newly registered, high-risk domains.
  • Input-Based Vulnerability Testing – Checks for unsafe handling of parameters that may expose the site to:
    • SQL Injection indicators
    • Reflected or stored XSS markers
    • Unsanitized user input embedded in HTML responses
  • SSL/TLS & Certificate Analysis – Validates the entire certificate chain, issuer, and expiration. Detects weak ciphers and outdated protocol support.
  • TLS Protocol Negotiation – Determines if the server allows insecure versions such as SSLv2, SSLv3, TLS 1.0, or deprecated ciphers.
  • Open Port Enumeration (Nmap) – Identifies open ports, banners, exposed services such as RDP/SSH, and potential attack surfaces.

Scoring & Risk Analysis

CyberSuite uses a weighted scoring model based on NIST standards (SP 800-40, 800-41, 800-52, and 800-177). Each component contributes to a final Security Posture Score that reflects the site's trustworthiness and technical risk.

The final output consolidates all scanner data — vulnerability indicators, certificate strength, protocol support, port exposure, domain intelligence, and phishing reputation — into a clean, readable summary inside the dashboard.

Conclusion

CyberSuite demonstrates a modern, modular approach to personal cybersecurity.
