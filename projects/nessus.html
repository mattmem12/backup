Part 1 - Vulnerability Validation

This project focused on confirming Nessus scan results for Bind Shell Backdoor, SSL/POODLE, and Apache Tomcat default credentials using Linux commands, Nmap scripts, and web browsing.

1. Bind Shell Backdoor

Ran a basic nessus scan on a machine vulnerable to a bind shell backdoor. I can confirm the Nessus results by the successful use of the standard Linux command id, which confirmed the vulnerability.

2. SSL/POODLE Validation

Ran a basic nessus scan on a site vulnerable to ssl version detection. Validated using Nmap scripts: nmap -sV --script ssl-poodle and nmap --script ssl-enum-ciphers. Both confirmed weak SSL/cipher issues.

3. Apache Tomcat Default Page

Ran a basic nessus scan on a site with an Apache vulnerablity and open port. Confirmed by opening http://10.0.2.6:8180/tomcat-docs/index.html in a browser. The default docs were accessible.

Key Learnings

  • How to run and interpret Nessus scans
  • Manually validate true positives vs false positives
  • Use Nmap scripts and Linux commands for verification
  • Understand CVSS scoring and remediation guidance

Part 2 - OS Audit & Web Scan Analysis

This project included an OS audit scan, host information scan, and web application scan across multiple hosts, highlighting vulnerabilities and risk assessments.

1. OS Audit Scan — Host 10.0.2.6

Total vulnerabilities: 275 (17 Critical, 88 High, 115 Medium, 9 Low, 46 Info). Medium and high vulnerabilities account for 87% of findings. Critical CVSS 10 vulnerabilities pose immediate risk.

Medium Example: MySQL outdated package (USN-671-1) Attack Vector: Network, Access Complexity: High, Authentication: Single, CVSS: 4.6
Fix: Update affected packages.

2. Host Enumeration — Host 10.0.2.4

6 informational findings, including database and port info. No actionable vulnerabilities detected.

3. Web Application Scan — Host 10.0.2.6

35 vulnerabilities: 1 Critical, 1 Medium, 33 Info. Medium Example: HTTP TRACE / TRACK enabled CVSS: 5.3, Network attack vector, Low complexity, No authentication
Fix: Disable TRACE and TRACK methods.

Key Learnings & Insights

  • Nessus helps identify, prioritize, and remediate vulnerabilities
  • Configuration audits show security gaps, compliance scans show adherence to standards
  • Combining host and web scans gives a comprehensive security overview
  • Proper remediation prioritization is critical based on CVSS and risk exposure
